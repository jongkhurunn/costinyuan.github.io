群晖DS224+ mihomo安装部署

步骤参考：https://www.ku0.cc/?p=407

下载安装包

在github MetaCubeX/mihomo仓库的release界面中找到mihomo-linux-amd64-compatible-v1.xx.x.gz二进制文件下载，解压

安装运行文件

创建一个../homes/mihomo文件夹

使用file station把mihomo-linux-amd64-compatible-v1.xx.x上传到mihomo文件夹中，使用ssh更改文件权限并移动到系统中

cd ../homes/mihomo
chmod +x mihomo-linux-amd64-compatible-v1.xx.x
mv mihomo-linux-amd64-compatible-v1.xx.x /usr/bin/mihomo

下载配置文件

  1. 下载国家ip数据库，通过file station放入mihomo文件夹

https://github.com/Dreamacro/maxmind-geoip/releases

  1. ssh下载配置文件（只有名字是'config.yaml'才会被识别）(..换成绝对路径)
wget -o  ../homes/mihomo/config.yaml "https://xxxxxx.xxx/api/v1/client/subscribe?token=xxxxxx&flag=meta"
  1. 尝试启动（..换成绝对路径）
/usr/bin/mihomo -d ../homes/mihomo/

如果没有端口号，检查配置文件里是不是没写

配置文件说明及参考

  1. 必须有端口号（port、socks或者直接用mixed-port）
  2. 必须有对外控制端口（external-controller）
  3. 为了使用metacubexd可视化控制，需要在mihomo文件夹中新建一个ui文件夹，并下载 ‘https://github.com/MetaCubeX/metacubexd/archive/refs/heads/gh-pages.zip’ 中的内容解压到ui文件夹中（external-ui参数替换成ui文件夹的绝对路径）
mixed-port: 7890 # HTTP(S) 和 SOCKS 代理混合端口

allow-lan: true
bind-address: "*" # 绑定 IP 地址，仅作用于 allow-lan 为 true，'*'表示所有地址
skip-auth-prefixes: # 设置跳过验证的 IP 段
  - 127.0.0.1/8
  - ::1/128
lan-allowed-ips: # 允许连接的 IP 地址段，仅作用于 allow-lan 为 true, 默认值为 0.0.0.0/0 和::/0
  - 0.0.0.0/0
  - ::/0
lan-disallowed-ips: # 禁止连接的 IP 地址段，黑名单优先级高于白名单，默认值为空
  - 192.168.0.3/32

mode: rule

#自定义 geodata url
geox-url:
  geoip: "https://fastly.jsdelivr.net/gh/MetaCubeX/meta-rules-dat@release/geoip.dat"
  geosite: "https://fastly.jsdelivr.net/gh/MetaCubeX/meta-rules-dat@release/geosite.dat"
  mmdb: "https://fastly.jsdelivr.net/gh/MetaCubeX/meta-rules-dat@release/geoip.metadb"

geo-auto-update: true # 是否自动更新 geodata
geo-update-interval: 360 # 更新间隔，单位：小时

log-level: debug

ipv6: true

external-controller: 0.0.0.0:9090 # RESTful API 监听地址
external-ui: ../mihomo/ui # 替换成绝对路径
external-ui-name: xd
external-ui-url: "https://github.com/MetaCubeX/metacubexd/archive/refs/heads/gh-pages.zip"

dns:
  enable: true
  listen: 0.0.0.0:53
  default-nameserver:
    - 223.5.5.5
    - 8.8.8.8
    - 2400:3200::1
    - 2001:4860:4860::8888
    - system
  nameserver:
    - 114.114.114.114 
    - 1.1.1.1
    - 2606:4700:4700::1111

打开群晖系统代理

控制面板-网络-常规-代理服务器，填写地址'127.0.0.1'，端口号就是配置里的'7890'

打开xd控制面板

在浏览器中输入'192.168.xx.xx:9090/ui'进入控制面板，地址是'192.168.xx.xx:7890'

添加自启动

前面步骤都成功后，就可以将mihoyo加入系统服务后台运行和开机自启了

创建文件 sudo vim /etc/systemd/system/mihomo.service，{..}替换成配置文件夹的绝对路径

[Unit]
Description=mihomo Daemon, Another Clash Kernel.
After=network.target NetworkManager.service systemd-networkd.service iwd.service

[Service]
Type=simple
LimitNPROC=500
LimitNOFILE=1000000
CapabilityBoundingSet=CAP_NET_ADMIN CAP_NET_RAW CAP_NET_BIND_SERVICE CAP_SYS_TIME CAP_SYS_PTRACE CAP_DAC_READ_SEARCH CAP_DAC_OVERRIDE
ExecStartPre=/usr/bin/sleep 1s
ExecStart=/usr/bin/mihomo -d {..}/mihomo
ExecReload=/bin/kill -HUP $MAINPID

[Install]
WantedBy=multi-user.target

重载systemd:

sudo systemctl daemon-reload

启用clash服务:

sudo systemctl start mihomo
sudo systemctl enable mihomo

使用以下命令检查 mihomo 的运行状况：

systemctl status mihomo

自动更新配置文件任务

  1. 在mihomo文件夹下创建脚本update.sh:
wget -O  /volume1/homes/clash/config.file "https://xxxxxx.xxx/api/v1/client/subscribe?token=xxxxxx&flag=meta"
#如果下载的配置文件不为空 重命名为config.yaml覆盖
if [ $(cat ../homes/mihomo/config.file|grep 7890|wc -l) -gt 0    ];then
/bin/mv ../homes/mihomo/config.file ../homes/mihomo/config.yaml
systemctl restart mihomo
fi
  1. 在‘控制面板-任务计划-新增-计划的任务-用户定义的脚本’中，选择好重复的时间，最后在‘任务设置’中写入：
/bin/bash {..}/mihomo/update.sh
